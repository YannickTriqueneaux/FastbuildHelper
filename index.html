FASTBuild Helper is an open-source template solution to facilite the integration of FASTBuild on an existing or a new C++ project

The idea

Personally a really like FASTBuild, this build tool do a very powerful job without any dependencies et complicated installations. But the problem is: if you want to use it, you must be an experimented programmer specially with compilation things and the final syntax is unreadable for a non-fastbuild-integrator. When I started to integrate FASTBuild in my last company I was beginer with compilers settings and the challenge was to port approximately 375 C++ projects (like VisualStudio projects), so the challenge was really tough but very interesting.

In order to succeed I imaginated an only one interface achieving the following goals:

  1. Must be readable and editable for a non-fastbuild-integrator.
  2. Should not expose any compilers things for those who don't understand them.
  3. Provide a multi-plateform interface.
  4. Should not create another language, the interface must uniquely be in FASTBuild's language.
  5. Should provide a format which can be generated easily.

How to use it

First, start by reading about the FASTBuild's language. The interface is readable without it but you may be handicapped and quickly do some mistakes. (see: FASTBuild language)

You have to know that this helper is not perfectly generic for all needs, this one provides a generic interface but in order to support your owns you will have to customize it according to your personnal requirements. In example: this one works with VisualStudio 2015 in Windows and GCC in Linux, also it will compile all *.cpp files inside your projects directories. But no worries, it's pretty easy to change those settings.

I advise you to start with a little "hello world" test. The helper provides an easy interface but not enough to stay in few lignes. Also you can rely on these Examples.
